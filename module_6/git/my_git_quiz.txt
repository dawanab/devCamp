GIT GET STARTED -------------->

1. What is the command to check which version of Git is installed?
    > git --version 

2. Initialize Git on the current folder:
    > git init 

3. Set the username for the current repository to "dawanab"
    > git config user.name "dawanab"

GIT NEW FILES ---------------->

4. Check the status of the Git
    > git status

GIT STAGING ENVIRONMENT ------>

5. Add index.html to the Staging Environment:
    > git add index.html

6. Shorthand command for staging all new, modified, and deleted files:
    > git add -A

GIT COMMIT ------------------>

7. Commit the changes to the current repository with the message "First Release!"
    > git commit -m "First release!"

8. Commit the updated files directly, skipping the staging environment:
    > git commit -a -m "New line added"

9. View the history of commits for the repository:
    git log

GIT HELP -------------------->

10. Show the possible options for the status command in command line:
    > git status -help 
    > git help --all  

GIT BRANCH ------------------->

11. Create a new branch called 'hello-world-images'
    > git branch hello-world-images

12. List the existing branches:
    > git branch

13. Move to the hello-world-images branch:
    > git checkout hello-world-images
    
14. Create, and move to a new branch with the name hello-you:
    > git checkout -b hello-you

GIT BRANCH MERGE ------------->

15. Merge the hello-you branch with the current branch:
    > git merge hello-you 

16. Remove the hello-you branch from the local repository:
    > git branch -d hello-you 

GIT REMOTE GET STARTED ------->
17. git remote add origin https://github.com/x/y.git

GIT PULL FROM REMOTE --------->
18. pull is a combination of:
    > fetch and then merge 

19. Get all the change history of the origin for this branch:
    > git fetch origin

20. Merge the current branch with the branch master, on origin:
    > git merge origin/master 

21. Update the current branch from its origin using a single command:
    > git pull origin 

22. push the current branch to its default remote origin:
    > git push origin

GIT PULL BRANCH FROM REMOTE ------------->
23. List all local and remote branches of the current Git.
    > git branch -a 

24. List only remote branches of the current Git:
    > branch -r 

GIT CLONE ------------------------------->
25. Clone the repository https://abc.com/x/y.git to your local Git:
    > git clone https://abc.com/x/y.git

26. Clone the repository https://abc.com/x/y.git to a folder named "newlife":
    > git clone https://abc.com/x/y.git newlife 

27. Rename the origin remote to upstream:
    > git remote rename origin upstream 

28. In .gitignore add a line to ignore all .temp files:
    > *.temp 

29. In .gitignore add a line to ignore all files in any directory named temp:
    > temp/ 

30. In .gitignore add a single line to ignore all files named temp1.log, temp2.log, and temp3.log:
    > temp?.log 

31. In .gitignore, ignore all .log files, except main.log:
    > *.log
      !main.log

GIT REVERT ------------------------------------>
32. Show the log of the repository, showing just 1 line per commit:
    > git log --oneline

33. revert the latest commit:
    > git revert HEAD

34. revert the latest commit, skipping the commit message editor:
    > git revert HEAD --no-edit 

35. revert the two last commits:
    > git revert HEAD~1
